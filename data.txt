
how can i create an api that when i call in any node js file it will give me the output of the data that i have saved on that api

You basically want one API server where data is stored, and any Node.js file can call that API to get the data. This is a very common backend pattern.

I'll show you the simple and practical way using Node.js + Express.

1. Create an API Server
First make a file called server.js

Install Express:

npm init -y
npm install express
Now create the API.

const express = require("express");
const app = express();

app.use(express.json());

// stored data
let users = [
    { id: 1, name: "Rahul", age: 20 },
    { id: 2, name: "Aman", age: 22 }
];

// API route
app.get("/users", (req, res) => {
    res.json(users);
});

// start server
app.listen(3000, () => {
    console.log("API running on port 3000");
});
Run it:

node server.js
Now your API is available at

http://localhost:3000/users
2. Call this API from another Node.js file
Create another file client.js

Install axios (for calling APIs)

npm install axios
Now call the API.

const axios = require("axios");

async function getUsers() {
    const response = await axios.get("http://localhost:3000/users");
    console.log(response.data);
}

getUsers();
Run:

node client.js
Output:

[
 { id:1, name:"Rahul", age:20 },
 { id:2, name:"Aman", age:22 }
]
3. If you want to SAVE data to the API
Add POST API

app.post("/users", (req,res)=>{
    const newUser = req.body;
    users.push(newUser);
    res.json({message:"User added"});
});
Then send data like this:

axios.post("http://localhost:3000/users",{
    id:3,
    name:"Karan",
    age:25
});

So any Node.js file, frontend, mobile app, or website can use that API.

Cannot GET /

Cannot GET / means your server is running, but there is no route defined for /.

Express is basically saying:

"You requested / but I don't know what to return."

This is normal if your only route is something like /data or /users.

Example
If your server code looks like this:

const express = require("express");
const app = express();

app.get("/data", (req,res)=>{
    res.send("This is my data API");
});

app.listen(3000, ()=>{
    console.log("Server running");
});
Then these URLs behave like:

http://localhost:3000/        ❌ Cannot GET /
http://localhost:3000/data    ✅ Works
Fix (Recommended)
Just add a home route.

app.get("/", (req,res)=>{
    res.send("Welcome to my API");
});
Full example:

const express = require("express");
const app = express();

app.get("/", (req,res)=>{
    res.send("Welcome to my API");
});

app.get("/data", (req,res)=>{
    res.json({name:"Rahul",age:20});
});

app.listen(3000, ()=>{
    console.log("Server running on port 3000");
});
Now these will work:

http://localhost:3000/
http://localhost:3000/data
Quick debugging rule
When you see:

Cannot GET /something
It means:

app.get("/something") is missing

can user access the data as string

Yes 👍 a user can access the data as a string. In Express, everything you send with res.send() becomes a string response in the browser.

Right now you are sending JSON:

res.json(data);
That sends structured JSON.

If you want the user to receive it as a plain string, convert it.

Method 1 — Send a normal string
app.get("/data", (req, res) => {
  res.send("This is my API data");
});
User will see in browser:

This is my API data
Method 2 — Convert your array to string
app.get("/data", (req, res) => {
  res.send(JSON.stringify(data));
});
Output:

[{"id":1,"name":"Rahul"},{"id":2,"name":"Aman"}]
Now it is pure string format.

Method 3 — Custom formatted string (better)
app.get("/data", (req, res) => {

  let result = "";

  data.forEach(user => {
    result += `ID: ${user.id}, Name: ${user.name}\n`;
  });

  res.send(result);
});
Output:

ID: 1, Name: Rahul
ID: 2, Name: Aman
